# DC-10.1 — Electron Entry Point Ownership

**Gate:** DC-10.1 Runtime Stabilization  
**Purpose:** Clarify which Electron entry point is canonical for the Desktop Control Center.

---

## Entry Points

| Entry | Path | Purpose |
|-------|------|---------|
| **Desktop Control Center** | `dist-electron/electron/desktop/main.js` | Canonical read-only operator cockpit. Single window, bounded preload, read-only IPC. This is the main entry used when running `electron .` or packaging for DC-11. |
| **Full app** | `dist-electron/electron/main.js` | Full Etsy Mentor app with BrowserWindow + BrowserView, gate7, DB, OpenAI, sessions. Use for full-feature development. Not the canonical packaged entry. |

---

## Canonical Runtime

The **Desktop Control Center** (`electron/desktop/main.ts`) is the canonical runtime for:

- Packaged desktop app (DC-11)
- `package.json` main field
- `npm run dev:electron` (after build)

---

## How to Run

| Command | Launches |
|---------|----------|
| `npm run dev:desktop` | Desktop Control Center (explicit path) |
| `npm run build` then `electron .` | Desktop Control Center (via package.json main) |
| `electron dist-electron/electron/main.js` | Full app (manual) |
